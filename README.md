# github-for-beginners

Beginner-friendly GitHub notes using **GitHub Desktop** (plus a few optional command-line checks).  
Simple setup + daily workflow for both non-code and code projects.

## Who it’s for

- Total beginners (never used Git/GitHub before)
- People using **GitHub Desktop** for non-code projects (docs, design files, etc.)
- Anyone who wants a clean, repeatable workflow

## How to use this repo (read order)

Start here:

1. `START-HERE.md` — the “what to do first” checklist

Then the guides (in order):

1. `guides/01-install-and-accounts.md`
2. `guides/02-ssh-keys-push-pull-mac.md`
3. `guides/03-ssh-keys-push-pull-win.md`
4. `guides/04-github-desktop-basics.md`
5. `guides/05-daily-workflow-non-code.md`
6. `guides/06-daily-workflow-code.md`
7. `guides/07-branches-collab-basics.md`
8. `guides/08-fixing-common-mistakes.md`

Reference pages:

- `glossary.md` — simple definitions (plain English)
- `resources.md` — useful links + later YouTube links
- `templates/commit-messages.md` — good commit message patterns
- `templates/issue-template.md`
- `templates/pull-request-template.md`

## Contributing (small + simple)

Quick rules:

- Keep it beginner-friendly (plain English, short steps)
- One topic per page
- Prefer checklists over long theory
- If you add a new guide, keep the numbering pattern

Best way to contribute:

1. Open an Issue (typo / suggestion / missing topic)
2. Or open a Pull Request with a small, focused change
